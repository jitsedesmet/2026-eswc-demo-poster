Composable SPARQL Parsing and Query Execution
with Traqula and Comunica
Jitse De Smet  ·  Ruben Taelman
⚠ Problem
  • SPARQL is fragmenting: 1.1, 1.2, GeoSPARQL, ADJUST, LATERAL — every variant needs different parsing logic
  • Monolithic parsers force long-lived forks that break upstream compatibility
  • Comunica's modular engine needs an equally composable language layer end-to-end
5+ SPARQL variants
26 rule deltas in 1.2
0 forks needed
💡 Compose your SPARQL parser and query engine on-the-fly — no fork, no rewrite, no breaking change.
⌨ Rule-based composition API
// Extend SPARQL 1.2 with LATERAL join
import { Sparql12ParserBuilder }
  from '@traqula/parser-sparql-1-2';
import { lateralRule }
  from '@traqula/rule-lateral';

const parser = new Sparql12ParserBuilder()
  .addRule(lateralRule) // ← extend!
  .buildParser();
✔
Language composability
Rules are named first-class objects
↳ add, replace or remove any rule
↳ no fork, no rewrite required
✔
Engine composability
Comunica actors via Components.js
↳ parser & engine always aligned
↳ features toggle end-to-end
⚡ Traqula capabilities
  • Full SPARQL 1.1 — parse generate transform
  • SPARQL 1.2 as incremental delta — 26 rule changes
  • ADJUST extension, compatible with Oxigraph
  • LATERAL join as an opt-in rule extension
  • Builder pattern — clone & extend any variant
  • Open algebraic model for new Comunica operations
  • actors adapt via Components.js
