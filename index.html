Composable SPARQL Parsing and Query Execution
with Traqula and Comunica
Jitse De Smet  ·  Ruben Taelman
⚠ Problem
  • SPARQL has diverged into many dialects: 1.1, 1.2, GeoSPARQL, ADJUST, LATERAL — each needs different parse logic
  • Monolithic parsers require long-lived forks that break upstream compatibility over time
  • Comunica's modular actor architecture needs an equally composable language layer to unlock new query features end-to-end
5+ SPARQL variants
26 rule deltas in 1.2
0 forks needed
💡 Compose your SPARQL parser and query engine on-the-fly — no fork, no rewrite, no breaking change.
⌨ Rule-based composition API
// Extend SPARQL 1.2 with LATERAL join
import { Sparql12ParserBuilder }
  from '@traqula/parser-sparql-1-2';
import { lateralRule }
  from '@traqula/rule-lateral';

const parser = new Sparql12ParserBuilder()
  .addRule(lateralRule) // ← extend!
  .buildParser();
✔
Language tooling composability

Query engine composability

⚡ Traqula capabilities
  • Full SPARQL 1.1 — parse generate transform
  • SPARQL 1.2 as an incremental delta — 26 rule changes
  • ADJUST function extension, compatible with Oxigraph
  • LATERAL join as an opt-in rule extension
  • Builder pattern — clone & extend any existing variant
  • Open algebraic model for new Comunica operations
  • actors adapt automatically via Components.js
