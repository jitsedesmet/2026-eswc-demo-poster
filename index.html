Demonstration of Composable SPARQL Parsing
and Query Execution with Traqula and Comunica
Motivation
  • SPARQL ecosystems are increasingly diverse (versions, dialects, and extensions)
  • Tools should be configurable incrementally instead of maintained as monoliths
  • Composable language tooling and query engines reduce integration effort
Modular parser construction and modular query execution can be composed live in one demo. 
# Traqula feature composition (demo)
base: SPARQL_1_1
features:
  - sparql12
  - adjust
  - lateral

# Generated artifacts
lexer, parser, generator,
transformerToAst, transformerFromAst
✔ Composable language tooling Rules are named and swappable ↳ Parsers/generators adapt incrementally
✔ Composable execution stack Engine configs adapt to selected features Functionality emerges when parser and engine align
Demo capabilities
  • Toggle SPARQL 1.2 support
  • Toggle built-in ADJUST
  • Toggle lateral operation
  • Inspect generated Traqula builders and transformers
  • Inspect generated Comunica Components.js configuration
  • Execute federated queries with the selected setup
Contributions
  • Interactive showcase of Traqula modularity
  • Live coupling of parser setup and query engine setup
  • Clear evidence of parser-engine composability in practice
