Demonstration of Composable SPARQL Parsing and Query Execution
with Traqula and Comunica
Jitse De Smet & Ruben Taelman · IDLab, Ghent University – imec
Motivation
  • SPARQL is increasingly diverse: SPARQL 1.1, SPARQL 1.2 (RDF-star), engine-specific extensions (ADJUST, LATERAL…)
  • Existing parsers are monolithic — extension requires long-term forks or complex conditional logic
  • Query engines must match parser capabilities — currently a manual, error-prone process
Goal
  • Parsing pipelines that can be composed and extended incrementally from named rule objects
  • Query engines configurable to match the chosen language features
Two Modular Frameworks
Traqula Modular SPARQL parsing, generation & transformation. Grammar rules are named objects — add, replace, or remove any rule without touching the rest of the system.
Modular knowledge graph query engine. Dependency injection via Components.js — configure exactly which actors handle each query operation.
🔗 Both frameworks are configured simultaneously from the same feature selections, ensuring parser and engine always match.
Compose any SPARQL dialect from named rules — parser, generator, transformer & engine stay in sync

Interactive Demonstration

Web client that configures Traqula and Comunica simultaneously. Start from a SPARQL 1.1 baseline and enable:

  • SPARQL 1.2 — RDF-star & reification
  • ADJUST built-in function
  • LATERAL operation

The system shows the resulting Traqula builder config and Comunica Components.js config, then executes federated SPARQL queries with the composed setup.
