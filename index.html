Composable SPARQL Parsing and Query Execution
with Traqula and Comunica
Motivation
  • SPARQL is diversifying: SPARQL 1.1, 1.2, GeoSPARQL, ADJUST, LATERAL — each requiring different parsing logic
  • Monolithic parsers cannot be extended without long-term forks, breaking upstream compatibility
  • Comunica's modular engine needs a matching composable language layer to unlock new query features
Compose SPARQL parsers, generators and query engines on-the-fly — no fork, no rewrite. 
// Extend SPARQL 1.2 with LATERAL join
import { Sparql12ParserBuilder }
  from '@traqula/parser-sparql-1-2';
import { lateralRule }
  from '@traqula/rule-lateral';

const parser =
  new Sparql12ParserBuilder()
    .addRule(lateralRule) // ← compose!
    .buildParser();
✔ Language tooling composability Rules are named first-class objects ↳ Add, replace or remove — no fork needed
✔ Query engine composability Comunica actors adapt via Components.js ↳ Parser and engine always stay aligned
Traqula features
  • Parse, generate & transform SPARQL 1.1
    • Baseline for all variants
  • SPARQL 1.2 as incremental extension
    • 26 grammar rule deltas
  • ADJUST built-in — compatible with Oxigraph
  • LATERAL join support as opt-in rule
  • Builder pattern: start any variant from existing one
  • Open algebraic model for new Comunica operations
